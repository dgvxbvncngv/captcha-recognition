部署运行手册 (Deployment & Runtime Manual)

1. 部署前准备与拓扑参数

本手册以标准的 Ubuntu Server 22.04 LTS (x86-64) 操作系统为例。以下部署配置已经过环境验证，适合单机生产环境的部署。

1.1 系统服务端口与通信规划

系统部署时，各组件通信端口及协议绑定规则如下，部署前需确保以下本地端口未被占用：

+------------------+         HTTP (80)         +------------------+
|    Client User   |-------------------------->|    Nginx Proxy   |
+------------------+                           +--------+---------+
                                                        |
                                                        | Unix Socket / TCP (127.0.0.1:8000)
                                                        v
+------------------+         TCP (6379)        +------------------+
|   Redis Server   |<--------------------------|   Gunicorn/Flask |
+------------------+                           +--------+---------+
                                                        |
                                                        | Local Disk I/O (File Path)
                                                        v
                                               +--------+---------+
                                               |    SQLite DB     |
                                               +------------------+

- Nginx 反向代理： 监听公网 80（HTTP）或 443（HTTPS）端口。
- Gunicorn WSGI 服务： 监听本地环回地址 127.0.0.1:8000（不对公网开放）。
- Redis 缓存服务： 监听 127.0.0.1:6379（开启保护模式，严禁公网监听）。
- SQLite3： 运行于进程内部，占用本地磁盘文件 /var/www/captcha-system/instance/database.db。

2. 基础软件依赖安装

在 Ubuntu 终端下依次执行以下系统指令，更新源并安装必要的运行时依赖包：

# 2.1 更新系统软件包源

sudo apt update && sudo apt upgrade -y

# 2.2 安装系统基础开发工具、Python虚拟环境工具及关系型数据库 CLI

sudo apt install -y git python3-pip python3-venv build-essential sqlite3

# 2.3 安装 Tesseract OCR 引擎及简体中文、英文语言包

sudo apt install -y tesseract-ocr tesseract-ocr-eng tesseract-ocr-chi-sim

# 2.4 安装高性能 Nginx 反向代理服务器与 Redis 缓存服务

sudo apt install -y nginx redis-server

3. 项目目录构建与虚拟环境配置

遵循最小权限原则，我们在 /var/www/ 目录下构建项目。

# 3.1 创建系统发布目录并配置权限（允许当前操作用户作为所有者进行代码部署）

sudo mkdir -p /var/www/captcha-system
sudo chown -R $USER:$USER /var/www/captcha-system

# 3.2 切换工作路径并拉取代码

cd /var/www/captcha-system

# 模拟拉取或拷贝代码至当前文件夹

# git clone <your_repository_url> .

# 3.3 创建 Python 虚拟隔离环境 (Virtualenv)

python3 -m venv venv

# 3.4 激活虚拟环境并升级 pip 依赖管理工具

source venv/bin/activate
pip install --upgrade pip

# 3.5 安装 Python 三方依赖矩阵包

pip install -r requirements.txt

4. 存储与数据持久层配置

4.1 SQLite3 数据库初始化与 WAL 并发模式配置

# 创建 SQLite 数据库常驻文件夹

mkdir -p /var/www/captcha-system/instance

# 初始化 SQLite 数据库，并强制开启 WAL（Write-Ahead Logging）高并发读写模式

sqlite3 /var/www/captcha-system/instance/database.db "PRAGMA journal_mode=WAL;"

# 验证 WAL 模式是否开启成功（返回值为 "wal" 即代表成功）

sqlite3 /var/www/captcha-system/instance/database.db "PRAGMA journal_mode;"

4.2 Redis 生产运行安全配置

编辑 Redis 默认配置文件，确保其在安全受控模式下运行：

sudo nano /etc/redis/redis.conf

关键配置参数核对与修改：

bind 127.0.0.1 -::1       # 仅绑定本地回环，禁止外网直连
protected-mode yes       # 开启保护模式
maxmemory 512mb          # 限制最大内存开销，防止榨干服务器物理内存
maxmemory-policy allkeys-lru # 采用 LRU 淘汰算法释放过期 Session

重启 Redis 服务以应用新配置：

sudo systemctl restart redis-server
sudo systemctl enable redis-server

5. 守护进程服务与反向代理部署

5.1 配置 systemd 控制 Gunicorn 运行守护

编写服务单元文件，以便系统对后端 Flask/Gunicorn 实例进行进程状态监控与自愈：

sudo nano /etc/systemd/system/captcha.service

写入以下系统服务单元定义（.service 文件）：

[Unit]
Description=Gunicorn instance to serve Captcha System API
After=network.target redis-server.service

[Service]
User=root
WorkingDirectory=/var/www/captcha-system
Environment="PATH=/var/www/captcha-system/venv/bin"

# -w 4: 启动4个工作线程；--timeout 120: 运行超时判定为120秒，防范模型悬挂进程

ExecStart=/var/www/captcha-system/venv/bin/gunicorn -w 4 -b 127.0.0.1:8000 app:app --timeout 120
Restart=always
PrivateTmp=true
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target

启动并激活 systemd 开机挂载：

sudo systemctl daemon-reload
sudo systemctl start captcha.service
sudo systemctl enable captcha.service

5.2 配置 Nginx 站点反向代理与物理图片映射

移除 Nginx 默认占位站点配置，并编写专属反向代理规则：

sudo rm -f /etc/nginx/sites-enabled/default
sudo nano /etc/nginx/sites-available/captcha

写入站点代理规则配置（Server Block）：

server {
    listen 80;
    server_name _; # 绑定域名或 IP

    client_max_body_size 2M; # 限制上传单张图片上限为 2MB

    # 静态安全响应头
    add_header X-Frame-Options "DENY";
    add_header X-Content-Type-Options "nosniff";

    # 1. 静态前端页面和单页应用路由
    location / {
        root /var/www/captcha-system/static;
        index index.html;
        try_files $uri $uri/ =404;
    }

    # 2. 核心 RESTful API 路由反向代理至后端的 Gunicorn
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 3. 物理存储图片 Nginx 直接高速映射过滤读取，剥离一切脚本执行权限
    location /static/uploads/ {
        alias /data/uploads/;
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}

挂载并使 Nginx 配置生效：

sudo ln -s /etc/nginx/sites-available/captcha /etc/nginx/sites-enabled/
sudo nginx -t # 验证配置语法
sudo systemctl restart nginx
sudo systemctl enable nginx

6. 系统容灾备份与运维操作

6.1 SQLite3 在线热备份命令

由于 SQLite 在运行期间可能存在 WAL 写入未落盘情况，不允许直接使用 cp 命令备份。必须使用 .backup 安全命令进行在线热备份：

# 执行数据库在线热备份

sqlite3 /var/www/captcha-system/instance/database.db ".backup '/var/www/captcha-system/instance/database_backup_$(date +%F).db'"

6.2 物理图片资源每日异地冷归档备份命令

# 使用 rsync 增量备份物理图片目录并打包压缩

rsync -avz /data/uploads/ /backup/captcha_images_$(date +%F)/

7. 生产环境故障运行诊断手册 (Troubleshooting)

| 异常现象                                              | 可能原因                                            | 诊断排查命令                             | 解决方案                                                                             |
| :---------------------------------------------------- | :-------------------------------------------------- | :--------------------------------------- | :----------------------------------------------------------------------------------- |
| **访问网页提示 HTTP 502 Bad Gateway**           | 后端 Gunicorn 进程挂起或未正常启动。                | `sudo systemctl status captcha`        | 运行`sudo systemctl restart captcha` 重启守护进程并查看 `/var/log/syslog`。      |
| **访问网页提示 HTTP 504 Gateway Timeout**       | 深度学习推理悬挂死锁或请求耗时超过系统阈值。        | `journalctl -u captcha.service -n 100` | 调大 Nginx`proxy_read_timeout`；在推理端显式检查 `torch.set_num_threads(1)`。    |
| **上传图片提示 HTTP 500 Internal Server Error** | 运行 Web 的用户无`/data/uploads` 目录的写入权限。 | `ls -ld /data/uploads`                 | 重新修正文件夹属主权限：`sudo chown -R www-data:www-data /data/uploads`。          |
| **登录提示网络超时或失败**                      | 本地 Redis 缓存进程因未设置最大内存而被 OOM 杀死。  | `sudo systemctl status redis-server`   | 修改配置文件限制 Redis 物理最大开销并重启：`sudo systemctl restart redis-server`。 |
