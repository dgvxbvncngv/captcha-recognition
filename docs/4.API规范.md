基于深度学习的验证码识别与安全分析系统

API 接口设计说明书 (API Specification Document)

1. 全局设计规范

1.1 基本配置

- API 基础路径 (Base URL)： https://`<api-domain>`/api/v1
- 通信协议： 生产环境必须使用安全超文本传输协议（HTTPS），基于 TLS 1.2 或 TLS 1.3 版本。
- 字符编码： 统一采用 UTF-8。
- 默认网关端口： HTTPS 使用标准 443 端口（本地调试使用 HTTP 5000 或 8000 端口）。

1.2 接口公共请求头 (Global Request Headers)

所有发往本系统的 API 请求，均应遵循以下 HTTP 请求头规范：

| 请求头            | 是否必填 | 推荐值/格式                                     | 说明                                               |
| :---------------- | :------- | :---------------------------------------------- | :------------------------------------------------- |
| `Content-Type`  | 是       | `application/json` 或 `multipart/form-data` | 声明请求载荷的媒体类型（MIME Type）                |
| `Accept`        | 否       | `application/json`                            | 客户端期望接收的数据格式                           |
| `Authorization` | 否       | `Bearer <token_string>`                       | 鉴权令牌。访问受保护资源（如历史记录、日志）时必填 |

1.3 统一响应报文格式

系统返回的所有响应报文，无论成功还是失败，均采用标准 JSON 结构包装：

{
  "code": 20000,
  "message": "success",
  "data": {}
}

- code：业务状态码（5 位数整数，详见第 5.2 节）。
- message：对该次业务执行状态的简要文本描述。
- data：业务返回的具体荷载对象（若无返回对象，该字段为 null）。

2. 鉴权与流控设计

2.1 鉴权机制 (Authorization)

1. 系统使用基于 Token 的无状态鉴权模式。
2. 用户成功调用 /auth/login 接口登录后，系统会在响应数据中返回一个哈希 Token。
3. 客户端后续访问受保护接口时，必须将该 Token 写入 HTTP 请求头中：Authorization: Bearer <token_string>。
4. 服务端从请求头中提取 Token 并向 Redis 发起校验，若 Token 不存在或已过期，阻断请求并返回 HTTP 401
   (Unauthenticated)。

2.2 跨域与流控响应头 (CORS & Rate Limiting Headers)

为防范恶意请求并保障接口的高可用，API 响应头中会携带跨域和流控控制参数：

Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, DELETE, OPTIONS
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 54
X-RateLimit-Reset: 1698128450

- X-RateLimit-Limit：客户端单周期（60秒）内允许的最大请求数。
- X-RateLimit-Remaining：当前限流周期内剩余的可请求次数。
- X-RateLimit-Reset：限流计数器清零并重新计算的时间戳。

3. 详细接口定义 (API Reference)

3.1 身份认证模块 (Authentication)

3.1.1 用户登录 (POST /auth/login)

- 说明： 验证用户账户，颁发身份标识 Token。
- 请求参数 (Body JSON)： | 字段名 | 类型 | 是否必填 | 示例值 | 说明 | | :--- | :--- | :--- | :---
  | :--- | | username | String | 是 | user_test | 用户名（450位字符） | | password |
  String | 是 | pass12345 | 明文密码（6128位字符） |
- 请求 Body 示例：

  {
  "username": "user_demo",
  "password": "mypassword123"
  }
- 成功响应 (HTTP 200)：

  {
  "code": 20000,
  "message": "Login successful",
  "data": {
  "token": "4a71d34b-8e2c-4f11-9a2c-bbddf701c456",
  "username": "user_demo",
  "role": "user"
  }
  }
- 错误响应（密码错误，HTTP 401）：

  {
  "code": 40101,
  "message": "Invalid username or password",
  "data": null
  }

3.2 核心推理模块 (Algorithm Inference)

3.2.1 验证码实时预测接口 (POST /predict)

- 说明： 上传一张验证码图片，进行预处理、深度学习模型推理以及对照基准计算。
- 请求头 (Headers)：

  - Content-Type: multipart/form-data
  - Authorization: Bearer <token_string> (可选)
- 请求参数 (Form Data)： | 参数名 | 类型 | 是否必填 | 示例值 | 说明 | | :--- | :--- | :--- | :---
  | :--- | | file | Binary | 是 | captcha.png | 物理二进制图像，大小不超过 2MB |
- 成功响应 (HTTP 200)：

  {
  "code": 20000,
  "message": "success",
  "data": {
  "record_id": 1542,
  "prediction": "8r4x",
  "confidence": 94.15,
  "elapsed_time_ms": 118,
  "tesseract_compare": "8r41",
  "timestamp": "2023-10-24 14:20:11"
  }
  }
- 错误响应（文件后缀不符，HTTP 400）：

  {
  "code": 40001,
  "message": "Invalid file format. Only JPG/PNG are allowed.",
  "data": null
  }

3.3 历史记录模块 (History Management)

3.3.1 分页查询历史识别记录 (GET /history)

- 说明： 检索当前用户上传的历史识别详情（管理员则可检索全局数据）。

  :--- | :--- | :--- | :--- | | page | Integer | 否 | 1 | 2 | 当前查询页码 | | limit
  | Integer | 否 | 10 | 20 | 每页返回的记录数 | | start_time | String | 否 | - |
  2023-10-01 | 筛选起始时间，格式 YYYY-MM-DD | | end_time | String | 否 | - | 2023-10-24
  | 筛选结束时间，格式 YYYY-MM-DD |
- 成功响应 (HTTP 200)：

  {
  "code": 20000,
  "message": "success",
  "data": {
  "total": 128,
  "page": 2,
  "limit": 10,
  "list": [

  "record_id": 1541,

  "predicted_text": "8r4x",
  "confidence": 94.15,
  "elapsed_time": 118,

  },
  {
  "record_id": 1540,
  "image_url": "/static/uploads/20231024/a12b3c4d.png",
  "predicted_text": "y7p2",
  "confidence": 88.42,
  "elapsed_time": 125,
  "created_at": "2023-10-24 13:10:05"
  }
  ]
  }
  }

3.3.2 删除特定历史记录 (DELETE /history/`<id>`)

- 说明： 物理删除某条特定识别历史，并同步在服务器磁盘上清除物理原图。
- 请求头 (Headers)： Authorization: Bearer <token_string>
- 路径参数 (Path Variable)：
  - id (Integer，必填)：待删除的记录自增 ID。
- 成功响应 (HTTP 200)：
  {
  "code": 20000,
  "message": "Record and corresponding file deleted successfully",
  "data": null
  }
- 错误响应（记录不存在，HTTP 404）：
  {
  "code": 40001,
  "message": "Record not found or access denied",
  "data": null
  }
