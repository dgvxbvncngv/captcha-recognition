| 架构层级              | 技术/组件选型           | 精确版本约束            | 选型技术考量                                                                                                           |

2. 软硬件资源与算力测算

2.1 算力开销估算 (Computational Benchmarking)

本课题采用的端到端 CRNN 神经网络，模型权重文件大小约为 18.5 MB，参数总量约合 4.6M。

- 在 CPU（2.2 GHz 主频，单线程）上，单次前向推理计算量（FLOPs）约为 0.32 GFLOPs。
- 测算得出：普通云服务器单核 CPU 的理论峰值算力约为 4.0 GFLOPs，因此单核 CPU 承载单次推理的计算耗时约在 80ms ~ 150ms
  之间，纯 CPU 推理方案技术可行，性能完全达标。

4. 磁盘目录权限与安全性设计

在生产环境中，应用服务应遵循**“最小权限原则”**（Least Privilege
Principle）。系统的磁盘目录结构、属主及文件访问控制列表（ACL）设计如下：

| 目录物理路径                          | 默认属主/属组 (Owner/Group) | POSIX 权限代码 | 设计安全目的                                                                               |
| :------------------------------------ | :-------------------------- | :------------- | :----------------------------------------------------------------------------------------- |
| `/var/www/captcha-system/`          | `root:root`               | `755`        | 保证系统核心代码目录对 Web  worker 为**只读**，防范远程代码执行漏洞（RCE）篡改源码。 |
| `/var/www/captcha-system/venv/`     | `root:root`               | `755`        | Python 虚拟环境依赖目录，只读保护，防止运行库被非法替换。                                  |
| `/var/www/captcha-system/models/`   | `root:root`               | `555`        | 深度学习模型权重目录，彻底屏蔽写入和修改权限，保证核心资产安全。                           |
| `/data/uploads/`                    | `www-data:www-data`       | `755`        | 验证码上传存储目录。必须可写，且其属主必须与运行 Nginx/Gunicorn 的用户保持一致。           |
| `/var/www/captcha-system/instance/` | `www-data:www-data`       | `700`        | SQLite 数据库文件所在目录。必须可写，权限`700` 隔离其他系统用户。                        |

5. 开发质量控制与静态分析门槛 (Linter Settings)

项目在开发集成阶段，统一采用 Flake8 作为代码质量分析器，Black 作为格式化程序，通过配置提交门槛，避免低质量代码并入仓库：

- .flake8 配置文件：
  [flake8]
  max-line-length = 79
  exclude = .git,__pycache__,venv,build,dist
  ignore = E203, W503
  max-complexity = 12
- 代码静态分析质量门槛：
  - 圈复杂度（Cyclomatic Complexity）最大限制： 12（任何函数的圈复杂度超过 12 必须进行代码重构拆分，降低逻辑混乱度）。
  - 静态质量扫描通过率： 100%（集成编译前必须无报错信息）。
