基于深度学习的验证码识别与安全分析系统

需求规格说明书 (Software Requirements Specification, SRS)

1. 引言

1.1 编写目的

本需求规格说明书旨在详细定义“基于深度学习的验证码识别与安全分析系统”的功能性与非功能性需求。本文档将作为系统设计、编码实现、系统集成以及软件测试的关键依据，确保研发团队、测试团队及项目相关方对系统的功能范围和质量指标达成一致。

1.2 系统背景

在现代网络安全防护中，验证码（CAPTCHA）作为区分人类操作与自动化机器脚本的基础防线，广泛应用于登录、注册、防刷等高频交互场景。然而，随着计算机视觉与深度学习技术的发展，传统字符验证码的防守能力正面临严峻挑战。

本系统旨在开发一个集成了计算机视觉预处理、深度学习端到端字符识别（CNN+RNN+CTC）、以及 Web
安全审计和历史趋势分析于一体的综合平台，用于定量评测验证码的安全强度，并为企业提供直观的验证码攻防效能审计。

1.3 专业术语与定义

- CRNN (Convolutional Recurrent Neural Network)： 卷积循环神经网络，一种结合了 CNN 特征提取与 RNN
  序列特征建模的端到端深度学习模型，适用于文本序列识别。
- CTC (Connectionist Temporal Classification)：
  联结主义时间分类损失函数，用于解决输入图像与目标序列（字符数量不固定、没有像素级对齐）之间的对齐问题。
- OTSU (大津法)： 一种自适应的图像二值化分割算法，根据图像的灰度直方图以类间方差最大化为原则选择阈值。
- QPS (Queries Per Second)： 每秒查询率，指系统在一秒内能够响应的请求数量。
- Levenshtein Distance (编辑距离)：
  一种衡量两个字符串相似度的方法，指由一个字符串转换成另一个字符串所需的最少单字符编辑操作（插入、删除或替换）次数。

核心业务处理数据流 (端到端验证码识别流)

1. 请求输入： 用户在前端界面选择或拖入一张验证码图片，点击提交；前端将图片转化为 multipart/form-data 流，携带用户 Token
   异步发送至 API 接口。
2. API 接收与前置拦截：

   - API 接口在 Redis 中校验请求频率，若超出限制（如每秒请求 > 5 次），拒绝服务并返回流控代码。
   - 校验文件大小（不超过 2MB）与后缀名，若合法则在 /data/uploads/ 目录下生成随机命名的本地文件。
3. 算法层预处理：

   - OpenCV 载入图像，利用 Luma 公式：Y = 0.299R + 0.587G + 0.114B 进行灰度化。
   - 使用中值滤波器去除孤立的高频噪声。
   - 应用 OTSU 大津法自适应二值化，分离字符区域与背景。
   - 强制调整图像维度至 160 \times 60 \times 1，并将其归一化至 [0, 1]。
4. 模型推理计算：

   - 加载模型单例，将预处理张量输入至 CRNN 特征图提取网络。
   - RNN 层分析序列字符的上下文逻辑，最后经由 CTC 贪婪解码器，提取出最终的文本序列，并根据前向传播各节点的 softmax
     概率乘积算出模型的综合置信度评分。
   - 多线程调度 Tesseract 模块生成对比组文本。
5. 持久化与返回： 将识别文本、置信度、耗时、原始图片相对路径、时间戳持久化写入 SQLite，同时更新 Redis 缓存中的今日统计总数。最终向前端返回
   JSON 报文。
6. 详细功能需求

3.1 图像上传与高级预处理模块 (FR-01)

FR-01-1：多格式图像解析

- 输入： 经由 Web 界面、API 接口提交的二进制图像流。
- 处理逻辑：
  1. 系统需检查魔术字（Magic Number），严格支持 image/png、image/jpeg、image/bmp。
  2. 若文件校验通过，将文件重命名为 uuid.ext。
- 输出： 返回服务器上的物理存储路径。
- 异常流：
  - 若上传的非图片文件（如文本、脚本），系统需捕获格式异常，拦截保存，并返回错误状态码 40001 及提示信息 “不支持的图像文件格式”。

FR-01-2：高抗噪图像二值化与去噪

- 输入： 原始图像矩阵。
- 处理逻辑：
  1. 灰度化： 转换彩色图为单通道灰度图。
  2. 去噪： 调用 OpenCV 中值滤波，卷积核大小设置为固定 3 \times 3，用于有效剥离验证码图像中的椒盐噪声和部分背景噪点。
  3. 自适应二值化： 采用大津法（OTSU）计算最佳全局阈值，对像素点进行背景剥离。
- 输出： 二值化后的灰度单通道图像矩阵。

FR-01-3：归一化与尺度变换

- 输入： 去噪后的二值化图像矩阵。
- 处理逻辑：
  1. 不考虑原图宽高比例，统一使用双线性插值算法将图像缩放为 160 \times 60 像素。
  2. 对像素值进行归一化操作：对每一个像素值 p，计算 p_{norm} = p / 255.0。
- 输出： 尺寸为 160 \times 60 \times 1，值区间为 [0.0, 1.0] 的浮点数矩阵。

3.2 深度学习特征提取与序列识别模块 (FR-02)

FR-02-1：卷积特征提取 (CNN)

- 输入： 维度为 [BatchSize, 1, 60, 160] 的归一化图像张量。
- 处理逻辑：
  1. 图像输入至 VGG/ResNet 特征提取网络。
  2. 经过多层卷积、ReLU 激活函数和池化层，逐步降低图像分辨率、扩大感受野，转换为表征图像纹理与边缘几何特征的特征图（Feature Map）。
- 输出： 提取特征后的高维张量。

FR-02-2：双向序列特征建模 (RNN)

- 输入： 卷积特征图序列。
- 处理逻辑：
  1. 将卷积特征按列切片，作为时间序列的输入帧。
  2. 输入双向长短期记忆网络（Bi-LSTM）中，自左向右和自右向左分别扫描序列，捕捉字符之间的上下文粘连及结构遮挡信息。
- 输出： 长度为序列步数的每个字符概率分布张量。

FR-02-3：端到端 CTC 解码与置信度量化

- 输入： 各时间步输出的 softmax 概率分布。
- 处理逻辑：
  1. 采用贪婪搜索（Greedy Search）解码，合并连续重复字符并剔除空白占位符（Blank）。
  2. 置信度评分计算公式：各解码出有效字符时间步的最大 Softmax 概率的连乘积：
     Confidence = \prod_{t=1}^{L} P(c_t | x) \times 100\%
- 输出： 解码后的验证码字符串文本，以及置信度评分（单精度浮点数，百分制表示，精确到小数点后两位）。

FR-02-4：基准对照识别

- 输入： 预处理后的图像路径。
- 处理逻辑： 调用 pytesseract.image_to_string，配置参数 --psm 7（单行纯文本识别模式）。
- 输出： Tesseract OCR 结果字符串。

3.3 实时推理与反馈模块 (FR-03)

FR-03-1：单实例异步推理

- 处理逻辑： 推理计算必须采用后台单实例加载模型。避免为每一次 HTTP 连接重复加载大模型参数导致内存和 CPU 暴涨。
- 功能表现： 页面提供异步表单接口，支持上传的同时无需整页重载。

FR-03-2：多维度推理结果反馈

- 输出数据：
  - 模型预测的字符串。
  - 模型计算得到的置信度评分。
  - 系统记录的预处理与推理的总耗时（单位：毫秒）。
  - Tesseract 识别文本，作为安全性评测的弱对照基线。

3.4 历史记录与统计报表模块 (FR-04)

FR-04-1：多维检索查询

- 输入参数： 用户 ID（自检索）、时间跨度（开始日期与结束日期）、置信度阈值上下限。
- 处理逻辑： 组装 SQL 语句对本地 t_history_record 表进行分页条件查询。
- 输出： 符合条件的结构化列表数据，包含每一次识别的历史图片相对路径、真实时间、预测文本、置信度等信息。

FR-04-2：可视化统计分析大屏

- 功能表现：
  1. 日请求负载折线图： 以天为单位展示近一周内的 API 累计调用次数。
  2. 置信度分布图（直方图）： 展示置信度区间（如
     \le 50\%，50\% - 75\%，75\% - 90\%，\ge 90\%）下的调用数量，直观反应模型对近期测试的验证码池的通过能力。
  3. 对比分析仪表盘： 统计并输出自研 CRNN 模型较 Tesseract OCR 的准确率提升差值。

3.5 用户管理与安全审计模块 (FR-05)

FR-05-1：基于角色的访问控制 (RBAC)

- 角色划分：
  - 普通用户： 仅可对自己名下的历史记录进行上传和查询。
  - 管理员： 可调阅全局系统审计日志，拥有全局历史数据看板的查询权限，并可冻结普通用户。

FR-05-2：操作审计日志追踪

- 处理逻辑： 每当用户进行“登录”、“发起识别”、“删除记录”或管理员“查阅日志”等敏感操作，系统必须同步启动独立线程，向 t_system_log
  写入一条不可篡改的日志记录。
- 日志记录字段： 包含操作用户、客户端 IP、浏览器 User-Agent、操作事件、执行状态、时间戳。

4. 详细非功能性需求

4.1 性能需求 (Performance Matrix)

下述性能指标基于第 5 节定义的标准 CPU 硬件平台测试得出：

| 指标项                     | 场景说明                       | 指标要求                                          |
| :------------------------- | :----------------------------- | :------------------------------------------------ |
| **单张推理平均延迟** | 无高并发，单用户上传测试       | 端到端平均计算时延$\le 400\text{ ms}$           |
| **高并发处理吞吐量** | 多用户持续压测，模型单实例加载 | 在 30 并发下，系统 QPS$\ge 15$ 次/秒            |
| **资源使用限额**     | 纯 CPU 推理，大批量并发运行    | Flask 单 worker 物理内存占用$\le 512\text{ MB}$ |
| **数据库响应时延**   | 历史记录单页查询（每页 10 条） | 数据库查询并反序列化 JSON$\le 50\text{ ms}$     |

4.2 准确率量化要求 (Accuracy Requirements)

模型对字符识别的精确性要求如下（完全匹配准确率的定义是指，整张验证码的所有字符必须与标签完全一致，少一个、多一个、错一个均算作识别失败）：

- 标准级验证码（无旋转、干扰线较少，长度 4-6 位）： 完全匹配准确率 \ge 92\%。
- 复杂级验证码（中等扭曲、带多重网格噪点、字符部分粘连）： 完全匹配准确率 \ge 75\%。
- 单个字符识别准确率（Character-level Accuracy）： 在复杂噪点下，单个字符级准确率 \ge 88\%。

4.3 安全防范需求 (Security & Isolation)

- API 接口防暴力穷举与 DDoS：
  - 采用令牌桶限流算法。
  - 限制单一用户/单一 IP 地址的验证码预测请求，每分钟上限为 60 次；高频调用返回 429 响应码。
- 静态路径隔离：
  - 上传到服务器的文件不能具有可执行权限，需统一执行 chmod 644 剥离执行属性，防止木马文件。
  - 禁止前端直连服务器绝对路径，图片通过 Nginx 静态别名映射输出（如映射 /data/uploads/ 到虚拟目录
    /static/uploads/）。
- 密码安全存储：
  - 用户数据库内严禁存储密码原文。
  - 采用加盐哈希算法（如使用 Python 社区的 scrypt 或 PBKDF2 算法，哈希长度不少于 128
    位）对密码进行处理，防范彩虹表撞库。

5. 数据定义与系统业务规则

5.1 输入格式规则

1. 文件扩展名检测白名单： 仅限 ['.png', '.jpg', '.jpeg', '.bmp']（大小写不敏感）。
2. 图像尺寸范围： 最小尺寸限制为 32 \times 32 像素，最大尺寸限制为 1000 \times 1000
   像素（过大的图片由于需要缩放，极易造成推理时延超时且对验证码而言无实际应用意义）。
3. 单文件上传限额： 文件物理体积最大限制为 2.0\text{ MB}。

5.2 业务审计规则（验证码难度自动分级）

系统将历史识别难度自动记录分级，用于企业做安全态势评估：

- 低风险级别： 模型识别置信度 \ge 90\%，表明当前验证码池基本对机器形同虚设，网站安全系数处于极低值。
- 中等风险级别： 模型识别置信度在 70\% - 90\% 之间，表明具有一定防守能力，但仍能被自动化工具部分突破。
- 高防御级别： 模型识别置信度 \le 70\% 且平均置信度持续走低，说明当前使用的验证码扭曲及噪点逻辑处于较好防御状态。

6. 运行环境与软硬件约束

6.1 硬件环境约束

- 宿主机最低 CPU 要求： Intel Core i5 或同性能服务器级处理器（x86-64 架构），主频 \ge 2.0\text{ GHz}。
- 内存空间： 物理内存空闲容量 \ge 4.0\text{ GB}。

6.2 软件环境约束

- 操作系统： 推荐运行于 Ubuntu 20.04 LTS 或更高版本的 Linux 生产环境。
- 解释器版本： Python 3.9 及以上（不向下兼容 Python 2.x 版本）。
- 数据库支持：
  - 关系型：标准 SQLite 3（无需配置独立数据库网络监听，单文件存储，部署开销小）。
  - 缓存型：Redis 6.x 及以上（确保 Redis 与 Flask 实例宿主机网络延迟 \le 5\text{ ms}）。
